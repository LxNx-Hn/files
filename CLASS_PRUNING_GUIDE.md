# Class Pruning Guide

## 목적
- 메인 실험 결과를 먼저 보존한 뒤 후속 분석으로 제거 후보 장르를 판단하기 위한 기준 정리
- 성능이 낮은 장르를 무조건 제거하지 않고 왜 어려운지 먼저 해석하기 위한 체크리스트 제공
- 최종적으로 축소 실험이 필요한 경우 일관된 기준으로 장르를 제외하기 위한 문서

## 기본 원칙
- 첫 번째 보고 결과는 15개 장르 전체 기준으로 유지
- 장르 제거는 메인 결과를 대체하지 않고 후속 분석으로만 수행
- 제거 기준은 단일 지표 하나가 아니라 여러 근거를 함께 사용
- 단순히 점수가 낮다는 이유만으로 바로 제외하지 않음

## 먼저 확인할 결과 파일
- `results/summary_report.txt`
- `results/summary.json`
- `results/plots/confusion_matrix_best.png`
- `results/plots/per_class_f1_best.png`
- `results/runs/<best_model>/misclassified_samples.csv`
- `results/runs/<best_model>/all_predictions.csv`

## 제거 후보 판단 기준

### 1. Per-class F1이 지속적으로 매우 낮은 장르
- `per_class_f1_best.png`에서 최하위권에 반복적으로 위치하는 장르 우선 확인
- 기준 예시
  - `F1 < 0.20`이면 강한 제거 후보
  - `0.20 <= F1 < 0.30`이면 보조 후보
- 한 모델에서만 낮은 장르는 보류
- 여러 모델에서 공통으로 낮은 장르를 우선 검토

### 2. 혼동 행렬에서 특정 장르와 거의 구분되지 않는 장르
- confusion matrix에서 특정 두 장르가 서로 지속적으로 뒤섞이면 병합 또는 제외 후보로 검토
- 예시
  - `Drama`와 `Romance`
  - `Horror`와 `Thriller`
  - `Mystery`와 `Thriller`
- 한 방향 혼동보다 양방향 혼동이 강한 경우를 더 중요하게 봄

### 3. 오분류 샘플을 봤을 때 실제로도 애매한 장르
- `misclassified_samples.csv`에서 해당 장르 샘플을 직접 확인
- 예측이 틀렸더라도 사람이 보기에 타당한 경우가 많으면 라벨 경계 문제로 판단
- 이런 경우 모델 문제보다 데이터 정의 문제일 가능성이 큼

### 4. 완화 평가와 엄격 평가 차이가 큰 장르
- `strict accuracy`는 낮지만 `Rlx@2`, `Rlx@3`가 높으면 다중장르 경계 문제일 가능성이 큼
- 이 경우는 바로 제거보다 장르 정의가 좁은지 먼저 해석
- 반대로 relaxed 지표에서도 계속 낮으면 진짜 어려운 장르일 가능성이 높음

### 5. 이미지와 텍스트 모두에서 약한 장르
- `image_only`와 `text_only` 둘 다 성능이 낮은 장르는 강한 제거 후보
- 한 모달에서만 약한 장르는 제거보다 모달 한계로 해석 가능
- 두 모달 모두 약한데 fusion에서도 개선이 없으면 제외 우선순위가 높음

## 제거 우선순위 산정 방법
- 아래 항목에 해당할 때마다 1점 부여
- `per-class F1 < 0.20`
- confusion matrix에서 특정 장르와 강한 양방향 혼동
- misclassified samples를 봤을 때 라벨 경계가 자주 애매함
- `Rlx@2`, `Rlx@3`를 봐도 개선폭이 작음
- `image_only`, `text_only`, fusion 모두에서 일관되게 약함

- 합계 해석
  - `4~5점`: 제거 1순위
  - `3점`: 제거 후보
  - `2점 이하`: 유지 권장

## 권장 절차
1. 15개 장르 기준 메인 결과 정리
2. 최고 성능 모델 기준 per-class F1 확인
3. confusion matrix에서 강한 혼동 쌍 확인
4. misclassified samples로 실제 샘플 점검
5. 제거 후보 2~4개 선정
6. 후보 제거 후 축소 버전 재실험
7. 축소 전후 성능 비교

## 추천 제거 방식
- 한 번에 너무 많이 제거하지 않음
- 1차 축소는 `2개` 또는 `3개` 장르 제거 권장
- 축소 후에도 의미가 유지되는지 확인
- 너무 유사한 장르만 남지 않도록 균형 유지

## 제거 전 체크 질문
- 이 장르는 실제로 모델이 못 푸는가
- 아니면 장르 경계 자체가 애매한가
- relaxed 평가에서는 회복되는가
- 단일모달과 멀티모달 모두에서 약한가
- 제거 후 과제가 지나치게 쉬워지지 않는가

## 발표용 해석 문장 예시
- 일부 장르는 장르 경계가 중첩되어 반복적인 혼동이 발생하였다
- 낮은 성능 장르는 단순 모델 한계보다 데이터 라벨 경계의 모호성과 관련될 가능성이 있었다
- 따라서 메인 결과는 15개 장르 기준으로 유지하고 후속 분석에서 제거 후보 장르를 별도로 검토하였다

## 권장 결론
- 메인 결과는 15개 장르 그대로 보고
- 제거 실험은 추가 분석으로 제시
- 제거 기준은 per-class F1, 혼동 행렬, 오분류 사례, 완화 평가 지표를 함께 사용
